# CS-300


    What was the problem you were solving in the projects for this course?
The projects for this course tasked us with creating a program that could parse comma-separated tokens (specifically university courses) from a textual input file and load these tokens into objects (courses), which were then to be stored within and handled by an overarching data structure.

    How did you approach the problem? Consider why data structures are important to understand.
We had several options for data structures to handle the parsed objects for these projects: vectors, hash tables, and binary trees. We studied all of them in preparation for completing this project. Ultimately, it was necessary to complete a runtime analysis of each data structure and make an informed decision as to which would best handle the gleaned data. Knowing what kind of data was to be handled, the size of that data, and the ease with which it could be sorted were all necessary precursors to properly selecting the best-suited data structure.

    How did you overcome any roadblocks you encountered while going through the activities or project?
As per usual, I consulted Github and Stack Overflow when I had problems. I also referenced the professor's feedback on my previous assignment submissions: his input helped me refine earlier pseudocode blocks that would not have functioned properly for the final assignment.

    How has your work on this project expanded your approach to designing software and developing programs?
I am much, much better at writing pseudocode now. Before this course, designing pseudocode that would handle program goals was far from being my strong suit. Now I am confident in that I can think conceptually about problems and translate my idea of how a program should function into modii which the programming language is capable of understanding and handling.

    How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?
Above all, my pseudocode-writing skils are now strong. This will be helpful in documenting and designing future projects.
